⚙️ Data Processing

The challenge uses LeRobot as the standard dataset format. Collected demonstrations must be converted to LeRobot v2.1 format before training.

1. Convert to LeRobot format

LeRobot v2.1 datasets are organized as episode-level parquet files and per-camera MP4 videos. Use the provided conversion script to transform your raw collected data into this structure. Refer to the LeRobot documentation for format details.

2. Choose task mode

Keep these settings consistent across data conversion and training:

Single-Arm

  • state_dim = 7
  • action_dim = 10
  • 1 wrist camera

Bimanual

  • state_dim = 20
  • action_dim = 20
  • 2 wrist cameras

3. Upload to Hugging Face

# Authenticate
hf auth login

# Use China mirror if needed
export HF_ENDPOINT=https://hf-mirror.com

# Upload the processed dataset
hf upload <your-hf-id>/<dataset-name> /path/to/processed_dataset . \
  --repo-type dataset

4. Download a dataset from Hugging Face

hf download <your-hf-id>/<dataset-name> \
  --repo-type dataset \
  --local-dir /path/to/local/dataset

🔍 Visualization

We use Rerun for dataset visualization — an open-source tool for logging and replaying multimodal robot data including camera streams, joint states and action trajectories.

Install Rerun

pip install rerun-sdk

Open a recorded .rrd file

rerun output.rrd

View in browser (no installation)

Rerun also provides a web viewer. Visit app.rerun.io and open a local .rrd file directly in the browser.

Make sure your rerun-sdk version matches the Rerun Viewer version — the data format is not stable across releases. See the official installation guide for details.

🚀 Deploy

openpi supports real-robot deployment via a WebSocket policy server. The robot client sends observations to the server and receives action chunks in return. Full details are in the openpi remote inference guide.

1. Start the policy server

python scripts/serve_policy.py \
  --config <your-config> \
  --checkpoint-dir checkpoints/<your-config>/<step>

The server listens on localhost:8000 by default.

2. Connect your robot client

On the robot side, use WebsocketClientPolicy from openpi to query the policy server. Pass the current observation dict and receive the predicted action chunk: 

from openpi_client import WebsocketClientPolicy

policy = WebsocketClientPolicy(host="localhost", port=8000)
action_chunk = policy.infer(obs_dict)
Before deploying, verify:
  • The observation dict keys and dimensions match those used during training.
  • The normalisation stats used at inference correspond to the same dataset used for training.
  • single_arm_mode and state configuration match the checkpoint.
